# Git 版本管理原子

## Overview

基于 git 提供配置文件的版本管理能力，支持变更追踪、版本回滚和历史查询。

---

## ADDED Requirements

### Requirement: Git 仓库初始化

系统 **MUST** 能够将指定目录初始化为 git 仓库。

#### Scenario: 初始化新的配置目录

- **WHEN** 用户指定一个目录作为配置目录
- **AND** 该目录不是 git 仓库
- **THEN** 系统必须初始化该目录为 git 仓库
- **AND** 创建初始提交

#### Scenario: 配置目录已是 git 仓库

- **WHEN** 用户指定一个目录作为配置目录
- **AND** 该目录已是 git 仓库
- **THEN** 系统必须复用现有仓库
- **AND** 不破坏现有提交历史

---

### Requirement: 自动提交变更

系统 **MUST** 在配置文件变更时自动创建提交。

#### Scenario: 配置文件修改后自动提交

- **WHEN** 托管目录中的配置文件被修改
- **THEN** 系统必须自动创建 git 提交
- **AND** 提交信息包含变更类型和影响的服务名称

#### Scenario: 批量变更合并提交

- **WHEN** 单次操作导致多个配置文件变更
- **THEN** 系统必须将所有变更合并为一个提交
- **AND** 提交信息列出所有变更的文件

---

### Requirement: 版本查询

系统 **MUST** 提供配置历史查询能力。

#### Scenario: 查询文件变更历史

- **GIVEN** 配置文件 `{file_path}` 存在于托管目录
- **WHEN** 用户请求该文件的变更历史
- **THEN** 系统必须返回该文件的提交列表
- **AND** 每条记录包含：提交哈希、时间、提交信息

#### Scenario: 查看特定版本的文件内容

- **GIVEN** 配置文件存在历史版本
- **WHEN** 用户指定提交哈希和文件路径
- **THEN** 系统必须返回该版本的文件内容

---

### Requirement: 版本回滚

系统 **MUST** 支持配置回滚到历史版本。

#### Scenario: 回滚单个文件

- **GIVEN** 配置文件存在历史版本
- **WHEN** 用户请求回滚特定文件到指定版本
- **THEN** 系统必须恢复文件内容到该版本
- **AND** 创建新提交记录回滚操作

#### Scenario: 回滚整个配置目录

- **WHEN** 用户请求回滚整个配置目录到指定提交
- **THEN** 系统必须恢复所有托管文件到该版本
- **AND** 保留回滚后的提交历史（不使用 reset --hard）

---

### Requirement: 变更差异查看

系统 **SHOULD** 提供版本间差异比较。

#### Scenario: 比较两个版本

- **WHEN** 用户指定两个提交哈希
- **THEN** 系统必须返回两版本间的 diff 内容
- **AND** diff 格式为统一 diff 格式
